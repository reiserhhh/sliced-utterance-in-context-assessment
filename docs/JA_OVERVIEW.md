## 1. これは何の検査法か

SUICA は、**玉を切らずに叩いて中身を当てる検査法**である。

一玉 = 一人が自然に書いたテキストの全体。質問紙のように「あなたは外向的
ですか」と尋ねる代わりに、その人が日々書いたものを多数の一切れ(128
トークンの切り身)に分け、決まった審査表で採点し、味を推定する。
質問紙(答え方を歪められる)と自由記述の投映法(解釈が主観に流れる)の
中間に、**凍結された・監査可能な・再現可能な**採点を置く——それが出発点の
野心だった。

## 2. 理論: 皮・果肉・種

スイカには皮があり、果肉があり、種がある。人のテキストも同じ構造を持つ。

**皮** = 話題・場面・投稿先。当初は「皮はノイズだから剥いで捨てる(統計的
に差し引く)」と考えた。**これは3通りの方法で反証された**——皮を差し引く
と人物の信号ごと消える。理由は単純で、**人は自分で皮を選ぶ**からだ。
どの畑に通うか自体がその人を語っている。この発見は後に定理になった
(F4: 皮の差し引きは媒介変数の調整であり、削れる信号量は正確に
Var(m)+2Cov(f,m)。割当が外生なら無害、という境界条件つき)。

そこで検査は3つの経路で聴く:

1. **皮選び** — どの畑に行くか。玉割り#1で外的妥当性の初認証を得た経路
   (政治・ニュースの畑選び → 開放性、r=+0.096)。
2. **果肉** — どの皮の下でも変わらない、その人の味。旗艦は一人称使用で、
   これも玉割り#1で認証された(→ 神経症傾向、r=+0.111)。
3. **皮なじみ** — 皮ごとに果肉の出方が変わる、その人固有の癖(if-then)。
   まだ暫定(2項目、よその玉比べを通過)。

そして重要な実測: **味は形式に宿る**。果肉系(純度ゲート合格)は4つ——
一人称・アポストロフィ省略・否定・罵り——全て**書き方の機械的な癖**で、
内容の語彙(何について書くか)はことごとく皮系に落ちた。個性は「何を
語るか」より「どう書くか」に住んでいる。これは著者識別の古典(機能語、
stylome)と一致する。

## 3. 種模様 — 「指紋」テーゼの操作化

ユーザーの直観:「世界に同じ二人は存在しない。因子をどれほど細かく
切っても、あの人の全てを表せない。しかしコーディングは可能で、集団から
離れている部分——残差——が個性だ。」

これは3つの数字になった:

- **因子で全ては表せない**: 審査表の項目群では、玉を識別できる信号の
  約32%が掬えない(埋め込み比較)。しかもその掬えない部分も玉ごとに安定。
- **しかしコーディングは可能**: 匿名の一切れ群から同じ玉を当てる
  目利き当ては AUC ~0.89。
- **残差が個性**: 並みの味(集団平均)を差し引いた後に残るその玉だけの
  並び=**種模様**は、半年置いても r=0.505(果肉系だけなら 0.684)で
  自分自身と一致する。よその玉比べの基準線は −0.02。

**二つと同じ種模様の玉はない——そして種模様は読める。** これが指紋テーゼの
現在の実証水準(開発畑内・探索級)。

## 4. 審査表 v4

- **物差し v4**: 辞書の重複語を捌いた版(20語の移動を全て理由付きで記録)。
  果肉系4項目の採点値は旧版と bit 同一——検証の鎖は切れていない。
- **項目の仕分け**(純度ゲート: クラス素の日持ち ≥0.15 かつ皮媒介率 <0.30):
  - 果肉系 4: 一人称、アポストロフィ省略(0.365=最強)、否定、罵り
  - 皮系 8: novelty、directive(境界)、話題語クラスタ群
  - 混合系 6: 二重表記で報告(TF 最良項目の positive-enthusiasm もここ)
  - 判定不能 1: tension(特性ではないという従来判定と整合)
- **成長ファミリー**: 文字 n-gram の共選択軸 21 本を A畑発見/B畑確認で
  追加(うち FORM 系 ~15: the/and/that の使い癖、句読点、語尾形態)。
  Mosteller-Wallace 以来の著者識別信号が、審査表に入ってくる。
- **配合表**: dev で決めて凍結したブレンド比率。次の玉割りの主予測子候補。

## 5. 比べてよい距離(L0-L4)

検証の過程で「どこまで比較が届くか」も測った。方向(◎最強、2つの育ちを
またいで T4 生存)> 順位(○減衰するが実在)> 同じ畑内のテーマまたぎ
(果肉系は実測係数つきで可)>> **育ちをまたいだレベル比較は禁止**
(F4 定理+PRED-4 の反証: 共変量の近道も FORM 座標の近道も存在しない)。
正当な道は2本だけ——同一人物に複数の育ちで書かせる設計(自前の畑)と、
育ちごとの並みの味で較正すること。この免許制は melon API がコードで
強制する(免許外の比較は例外を投げる)。

## 6. 蔵の状態

- **取り置き玉**: 残り1玉(+Essays 半玉は未接触)。玉割り#1(2026-07-07)
  は封印状どおり実行され、7項目中2項目が認証(上記の2経路)、総合判定は
  不合格として**そのまま記録**した。再分析はしない——それがこの検査法の
  存在意義だから。
- **封印状**: 公開の蔵(GitHub)に貼付済み。誰でも検証できる。
- **検品**: 11回。採点者(AI)のミスを機械的9件+解釈の誇張4件=13件捕捉。
  うち1回は玉の名札の取り違え(クラスタの再学習)という重大品だった。
  **検品の記録自体がこの検査法の一部**である。

## 7. これから

1. **自前の畑**(SUICA-native corpus): 同一人物 × 自由畑/指定畑 ×
   2セッション × 日英。状態の時間スケール・対話・育ちまたぎ比較・日本語、
   4つの宿題を一つの畑で閉じる設計書は書き上がっている。
2. **種まき**: FORM 共選択 15 軸の純度ゲート通過と審査表への正式編入。
3. **次の玉割り**: 配合表を主予測子に、検出力 ≥90% を確認してから、
   最後の一玉を割る。
4. **時間の蔵**: 未来のテキストへの予測を先に封印する仕組み(封印#1は
   バーが甘いと検品されたので、封印#2は採点器を完全実装してから、
   本当に不確実な予測だけを賭ける)。
