# Sliced Utterance In-Context Assessment(SUICA): 自発的言語からのパーソナリティ・アセスメントのための反証優先フレームワーク

## 要旨

パーソナリティ・アセスメントは長らく、自己報告の標準化と、遂行型
(performance-based)測度の自発的行動の豊かさとを兼ね備えた一つの検査を
求めてきた。本論文は SUICA を提示する。日常の文章を、凍結された機械的
採点をもつ遂行型検査として扱う枠組みである: 自発的テキストを、本人が
選んだ文脈の内部で多数の小さな発話に切り分け〔一切れ〕、三つのチャネル
——文脈選択〔皮選び〕、文脈不変のスタイル基底〔果肉〕、人×文脈の
シグネチャ〔皮なじみ〕——で読む。枠組みの土台は一つの反証である:
話題・場面の分散を統計的に除去するという分野の既定慣行は、文脈が自己
選択される限り人物信号を削除することを、実証的に、次いで代数的に示す
(除去される再検査共分散は選択媒介シェアに等しい)。文脈は減算ではなく
設計で扱わねばならない。妥当性ガバナンスは内蔵されている: バージョン
固定の採点器〔物差し〕、ハッシュ封印された一回限りの事前登録〔封印状〕
を伴うデータ階層、モデル自身の仮定を破った生成世界に対して免許を得た
推定量、段階的な比較ライセンス〔比べてよい距離〕、そして11ラウンドの
監査で13件の構築者エラーを捕捉・修正した敵対的な構築者/監査者
プロトコル〔検品〕。公開の Reddit・エッセイコーパスでの完全な実働構築
(開発層 N = 3,183)では、特性級の信号は形式習慣(代名詞・正書法・否定)
に集中し、内容語彙は文脈選択に乗ることが示された。封印済みの確認的
開封〔玉割り〕は二つのチャネル予測を確認した(一人称率→神経症傾向
r = .111、政治的場面選択→開放性 r = .096)一方、事前登録されたオムニバス
規則は不合格だった(2/7)——全て包み隠さず報告する。コード・
マニフェスト・封印済み事前登録は公開されている。

**キーワード:** パーソナリティ・アセスメント、自然言語、心理測定、
事前登録、開語彙分析、個人語(idiolect)

## 1. 序論

言語ベース・アセスメントの10年は、パーソナリティが日常の文章に利用可能
な痕跡を残すことを確立した(Pennebaker & King, 1999; Schwartz et al.,
2013; Park et al., 2015; Eichstaedt et al., 2021; Boyd & Schwartz, 2021)。
しかし、この分野の最良の予測モデルは、その精度を二つの負債で購って
いる。第一は解釈可能性: 開語彙モデルや埋め込みモデルは、項目単位で監査
できず、再学習なしに改訂できず、測定される本人に説明できない、数千の
不透明な特徴を生む。第二の負債はより静かで、我々の主張では概念的で
ある: 話題や場面をノイズ分散として扱う、ほぼ普遍的な慣行だ。トピック
モデルであれ、コーパス層化であれ、内容の残差化であれ、既定の
パイプラインは「本人が何について書くと選んだか」を統制した上で人物を
探す。

本論文は一つの反証から始まる。1,760万件の Reddit コメントのコーパスで、
我々はこのノイズ扱いを最強の形で試みた——素朴な条件平均中心化、縮小
つき部分中心化、交差適合で標本外推定した二元固定効果中心化——そして
**すべての変種が、すべてのスタイル測度の分離機会安定性を低下させた**
(プール差 Δ = −.091、95% CI [−.102, −.080]; 交差適合固定効果変種で
−.05〜−.07)。理由は統計的不運ではなく構造である: 文脈が自己選択される
とき、文脈は人物→テキスト効果の**媒介変数**である。媒介変数を条件づけ
れば媒介された信号は除去される。我々はその恒等式(除去される再検査
共分散 = Var(m) + 2Cov(f, m)。m は本人の文脈ミックス項、f は文脈不変
基底)、その境界条件(文脈が外生的に割り当てられる場合——課題作文——
に限り中心化は無害)、そして強い人物-文脈結合下での二次的悪化を導出
する。分野が洗浄と呼んできたものは、自己選択の下では削除である。

建設的帰結は、スイカにちなんで**皮モデル**と呼ぶ再枠組み化である: 話題
と場面は捨てる包装ではなく、人物信号を運ぶ皮である。なぜなら人は自分の
皮を選ぶからだ。テキスト検査は三つのチャネルを読むべきである: (C1)
文脈の**選択**それ自体——どの場面を、どの話題を、どの割合で。(C2)
スタイル**基底**——皮から皮へ渡っても生き残る習慣。(C3) 人×文脈の
**シグネチャ**——状況クラスを横断する行動の if-then パターン、行動
シグネチャ(Mischel & Shoda, 1995)のテキスト版子孫。文脈分散は設計
(皮の固定または標本化)で統制し、統計的減算では決して統制しない。

この野心には、計算論的転回に先立つアセスメント伝統の系譜がある。自己
報告inventoryは標準化を、項目の透明性・反応スタイル・内省的アクセスの
限界という代価で買った。遂行型の伝統(投映法以降)は代わりに自発的
行動へ手を伸ばし、採点の主観性と信頼性論争で支払った(Meyer et al.,
2001; Mihura et al., 2013)。我々の主張では、蓄積された日常の文章こそ、
この伝統が探していた遂行標本である: 豊富で、生態学的に産出され、項目
単位のリハーサルが不可能で——しかも投映法プロトコルと違い、**凍結された
機械的規則で採点でき、その全カウントが監査可能**である。SUICA の出発点
の野心はまさにこの統合である: 質問紙の標準化を、遂行型伝統の自発的
行動に適用すること。しかし枠組みは安価である。我々がアセスメント
共同体に主として提供するのは、この枠組みをあらゆる関節で反証可能に
した検証機構である:

1. **凍結された解釈可能採点。** 全スコアは固定辞書と固定文脈地図からの
   カウントで、バージョン固定され、等価性ゲートを持つ(凍結スコアを
   わずかでも変える採点器改訂は検証チェーンを再起動する——主張ではなく
   測定される)。
2. **推定量の wrong-world 免許。** 主張に使う全推定対象は、モデル自身の
   仮定を一つずつ破った生成世界の中で、理論が真の場合と偽の場合を区別
   できることをまず示さねばならない。この方法で2件の免許失敗が捕捉され、
   機構つきの記録済み FAIL として報告される。
3. **封印付き階層データガバナンス。** 開発ラベルは確認ラベルから隔離
   される。確認分析は事前登録され、公開リポジトリにハッシュ封印され、
   敵対的事前監査でスクリプト自身のハッシュも固定された上で、正確に
   一度だけ実行された。開封は逐語的に報告する: オムニバス成功規則は
   **不合格**(7仮説中2つが BH-FDR q < .05; 規則は4を要求)、一方で
   確認された2つは枠組みの中核2チャネルそのものである。
4. **敵対的な構築者/監査者プロトコル。** 全ての実質的結果は、台帳に
   入る前に、完全な再計算権を持つ独立監査者に攻撃された。11ラウンドで
   監査は13件の実エラーを捕捉した——自己推定リーク、最適化器の境界
   崩壊、推定対象の非対称、クラスタリングの黙った再学習による構成概念
   同定の取り違え、解釈の誇張——各々が修正とともに記録されている。
   我々はこのプロトコルとそのエラー分類法を、計算論的測定研究のための
   再利用可能な再現性手法として提案する。

さらに二つの貢献が枠組みから帰結する。全構成概念のチャネル分解は、
特性級の文脈不変信号が**形式**習慣——代名詞使用、正書法習慣
(アポストロフィ省略)、否定、罵り——に集中し、内容語彙の見かけの安定
性は文脈選択に負うことを示す。これは法言語学的スタイル計測の知見
(個性は形式の共選択に宿る; van Halteren et al., 2005; Wright, 2017)と
一致し、確認段階で最も明確に失敗した仮説を事前に防いだはずの入場ゲート
(果肉純度基準)を与える。また残差個性層は、いかなる因子リストも人を
汲み尽くさないという個性記述的直観(Allport, 1937; Molenaar, 2004)を
操作化する: 規範プロファイル〔並みの味〕を引いた後に残る偏差パターン
〔種模様〕は月単位で安定であり(プロファイル中央値 r = .505、他人帰無
〔よその玉比べ〕−.02; 形式構成概念のみで .684)——因子構造の外にある
測定可能な個性を、Furr(2008)のプロファイル分解の規律で定量する。

以下、枠組みと形式的結果(第2節)、検証アーキテクチャ(第3節)、反証
系列と確認を含む実働構築(第4-5節)、そして免許の限界——この検査が
許す比較と許さない比較——と設計済みの次研究(第6節)を提示する。

## 2. 皮の枠組みと形式的結果

### 2.1 モデルとチャネル

著者 u、テキスト一切れ i、その一切れが産出された文脈(皮)r(i) について

> y_ui = f_u + b_r(i) + γ_u,r(i) + e_ui,  r(i) ~ π_u

と書く。f_u は文脈不変のスタイル基底〔果肉〕、b_r は文脈のスタイル
オフセット、γ_u,r は人×文脈交互作用、e_ui は一切れnoise、π_u は本人の
文脈分布である。実質的な公約は最後の一句だけである: **文脈は選ばれる**。
そして選択分布 π_u 自体が人物の安定な性質である。三つの測定チャネルが
従う。C1(選択): 母集団に対する π_u——どの場面を、どの割合で。C2
(基底): f_u——文脈を渡っても生き残る習慣。C3(シグネチャ): γ_u,r の
構造——状況クラスを横断する if-then パターン(Mischel & Shoda, 1995 の
テキスト版子孫)。文脈にはレジーム〔育ち〕がある: 自由(書き手が場面も
話題も選ぶ; SNS)、割当(実験者がお題を固定; 古典的エッセイコーパス)、
ドメイン固定(外部ドメインがマクロ話題を固定; 専門コミュニティ)。
どのチャネルが測れるかはレジームに依存する: 設計で皮を固定すれば C1 は
沈黙し C2 は純化される。自由収集は C1 を得るが、あらゆる生スコアに皮が
混ざる。

### 2.2 六つの形式的結果

証明とシミュレーション検証は補足資料 A に置き、ここでは実務を変える
内容を述べる。(F1) 生のユーザー平均は合成量である:
E[ȳ_u] = f_u + m_u、m_u = Σ_r π_ur·b_r。π_u が人物安定である以上、
文脈ミックス項 m_u も人物安定である——生の「スタイル」スコアは果肉
**プラス**凍結された文脈選択を測る。(F2) 共有文脈セットでの再検査は
両者を分離できない: 共有セット再検査共分散は Var(f) + Var(m) +
2Cov(f, m) に等しい。f ≡ 0 の生成世界——個人スタイルが皆無——でも、
生の分離機会再検査は選択だけから r = .64 を読む。安定性それ自体は
スタイル特性の証拠ではない。この一事が、テキストの「信頼性」の読み方
を再編する。(F3) 文脈を素な集合に分割すればミックス-ミックス項は消える
(文脈効果が独立で、選択がそれと非結合のとき)。人物-選択結合は交差項
として生き残るので、素集合推定は果肉の上界である。クラス素変種は
カテゴリ水準の媒介も除去する(その免許世界では、文脈効果がクラス内で
相関するとき条件水準の分割は果肉を +.106 過大に読み、クラス素の腕は
植え込んだ果肉分散を許容誤差内で読む)。(F4) **中心化定理。** 文脈平均の
統計的除去——分野既定の「話題統制」——は媒介変数調整である。選択と
効果の独立の下で、それは再検査共分散を正確に Var(m) + 2Cov(f, m) だけ
除去し(オラクル・シミュレーション: 予測 −1.335 に対し観測 −1.332)、
強い人物-文脈結合の下ではそれ以上を除去する。推定された文脈効果が、
その文脈を選んだ人々の果肉を吸収するからである(推定効果を真の効果に
回帰した傾きは結合 κ = 0.7 で 1.40; 観測除去は一次予測 −1.294 に対し
−1.583 に達する)。境界条件は正確である: 外生的に割り当てられた文脈
ではミックス項が消え、中心化は無害である。話題分散の除去が洗浄なのは
割当設計のみであり、自己選択の下では削除である。(F5) 文脈異質性の精度
コストは構成概念の文脈負荷で重みづけられる——場面ミックスのエントロピー
は、構成概念の分散のうち文脈に乗る割合に比例して信頼性を削る。(F6)
モデルは過剰識別する: 機会・レジスタ横断の移送は √(rel₁·rel₂) で
上界され、独立な媒介推定量は一致せねばならず、その不一致は組み込みの
誤特定警報である(警報は我々自身のデータで2回発火し、該当シェアは点
ではなく帯で報告される)。

### 2.3 検査設計への帰結

三つの規則が従う。文脈は設計で統制し、減算では決して統制しない(F4)。
構成概念は純度検査——クラス素安定性が実質的で、選択媒介シェアが小さい
——の後にのみスタイル特性と呼びうる。F1 が、生の安定性は二チャネルを
混同することを保証するからである。そして全スコアは比較ライセンスを
帯びる: 文脈内モニタリング、テーマ横断、レジスタ横断、レジーム横断、
言語横断の各比較は証拠によって別々に免許され、レジーム横断の**レベル**
比較は端的に禁止される(第6節)。この禁止は公開ソフトウェアが強制する
——数値を返す代わりに例外を投げる。

## 3. 検証アーキテクチャ

枠組みの信用は、それを殺せたはずの機構の分だけである。開発中に各々
最低1件の実エラーを捕捉した五つの部品が、計算論的アセスメントのための
再利用可能なガバナンス標準として提案するものを構成する。

**等価性ゲート付き凍結採点。** 全スコアはバージョン固定辞書と固定文脈
地図からのカウントであり、モデル重みも fine-tuning もない。採点器改訂
は等価性を証明するか、検証チェーンを再起動せねばならない: v3.1 改訂
(アポストロフィ正規化)は見出し値を .002 超えては変えず採用された。
v4 改訂(辞書間で共有される20語の排他化、各再割当は理由つきで記録)は
凍結構成概念スコアを bit 同一に保ちつつ、広い特徴空間から製造された
カテゴリ間共分散を除去した。

**封印付き階層データ。** 開発ユーザー(N = 8,678)は確認ラベルから恒久
的に分離される。開発層アンカーラベルは方向づけにのみ使われ、全接触が
事前登録自体に開示される。確認層——Big Five ラベル付き1,401名、二重
ラベル375名、エッセイコーパスの保留半分——は、公開リポジトリの初期
コミットハッシュを封印とする事前登録の背後に封じられた: 仮説、方向、
適格性規則、多重性補正、成功規則は、確認ラベルが一つでも読まれる前に、
公開の場で固定され、開封予算はプロジェクト生涯で2回である。開封#1
(第5節)は、敵対的事前監査で自身のハッシュを固定されたスクリプトに
より一度だけ実行され、結果の如何にかかわらず完全に報告される。

**台帳優位の証拠階層。** 全主張は公開の主張台帳に、四階層——探索的・
同一データ事前公約・ホールドアウト・封印確認——のいずれかで存在し、
本文は台帳行が支持するより強い言葉を使えない。台帳は撤回と修正を
その場に記録する。本稿の全数値は台帳行とその一次アーティファクトに
遡る。

**推定量の wrong-world 免許。** 推定量の出力が主張を支持しうる前に、
モデルの仮定を一つずつ破った生成世界の中で、枠組みが真の場合と偽の
場合を見分けられることを示さねばならない。九つの世界が走った(選択
帰無、果肉帰無、回収グリッド、クラス相関文脈、結合選択、中心化位相図、
時間スケール混同、シグネチャ帰無、リーク限定)。七つが合格。二つは
単一機構——強結合下の推定文脈効果の構成バイアス——を共有する記録済み
FAIL を生み、該当推定量は点推定量ではなく宣言された上界に転換された。
スイートは自分自身の欠陥も捕捉し(タイに不安全な検定統計量が、タイの
多い帰無世界で信号を捏造)、「ハーネスは凍結推定量の規約を厳密に再現
せねばならない」という恒久規則を生んだ。

**敵対的な構築者/監査者プロトコル。** 全ての実質的結果は、実行前に
固定された基準の下、台帳に入る前に、完全な再計算権を持つ独立監査者に
攻撃された。11ラウンドで監査は13件の実エラーを捕捉した: 機械的9件——
中心化「救済」の自己推定リーク、偽収束1件を含む最適化器境界崩壊、判定
を反転させていた推定対象非対称、クラスタリングを transport せず黙って
再学習したことによる構成概念同定の取り違え——と、解釈の誇張4件——
